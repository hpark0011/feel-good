## Usage Notes

- **Trend** requires >= 3 sessions with scores for the same dimension. Use `—` for insufficient data.
- **Evidence** must reference specific signals from extraction (tool counts, keywords, ratios), not vague impressions.
- **Proposed updates** use diff-style blocks so the developer can review exact additions.
- **Anti-patterns** should map to specific items in the rubric (e.g., "investigation_ratio < 1.0 in 3/5 sessions").
- Keep per-session breakdowns concise — 1-2 sentences of evidence per dimension, not paragraphs.
